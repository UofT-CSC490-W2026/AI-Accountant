
Overall:
- For MVP/SLC purposes, the project would be a simple docker/podman-compose monolith, centered on
  a FastAPI + Postgres stack. (It'll be easy to switch to Kube if & when we need it.)

Fast API: (lives in the backend/ directory)
- On the Python side of things, we'll be using uv for managing packages and environments (which is 
  better for production, and you don't have to type commands to explicitly enter-exit virtual
  environments).

DB:
- The main DB is PostgreSQL. For MVP purposes, data is persisted through volumes.
- For the interaction between backend and DB, the stack is:
  FastAPI + Pydantic -> SQLAlchemy ORM + PsycoPG -> Alembic (migration management)
